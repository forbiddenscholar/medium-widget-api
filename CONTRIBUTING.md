# Contributing

Thankyou! for considering to contribute to this project.
Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Install dependencies: `npm install`
4. Create a new branch for your feature: `git checkout -b feature/your-feature-name`

## Development

To test changes, deploy to Vercel with a preview branch or set up a testing environment.

## Making Changes

- Keep changes focused and minimal
- Follow existing code style
- Test your changes work correctly
- Update README.md if adding features

## What is Accepted

- Bug fixes
- Performance improvements
- Error handling enhancements
- Documentation improvements
- SVG styling improvements
- Dark mode refinements

## What is not Accepted

- Breaking changes to the API route
- Unnecessary dependencies
- Unrelated features

## Submitting Changes

1. Push your branch to your fork
2. Open a Pull Request on GitHub
3. Describe your changes clearly
4. Wait for review

## Code Style

- Maintain readability
- Add comments for complex logic
- Use descriptive variable names
- Follow existing patterns in the code

## Issues

Found a bug? Want a feature? Open an issue on GitHub with:

- Clear description
- Steps to reproduce (for bugs)
- Expected vs actual behavior

## Questions?

Open an issue and tag it as a question.
